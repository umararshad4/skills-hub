# Design Consistency

Inspect the existing UI before changing it.

## Rules

- Preserve the current visual language unless a redesign is requested.
- Match existing spacing, typography, radius, shadows, density, icons, and component variants.
- Do not introduce one-off colors or layout patterns.
- If the user asks for a narrow change, do only that change.
- For meaningful UI changes, verify desktop and mobile layout.

## Report

State what visual system was followed, what changed, and whether any responsive check was run.
