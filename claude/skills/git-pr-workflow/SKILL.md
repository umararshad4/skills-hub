# Git PR Workflow

Always inspect actual branch and diff before writing commit or PR text.

## Rules

- Do not mention AI tools or agents in commit/PR text.
- Commit messages and PR bodies must describe actual diff content.
- Run risk-based checks before PR readiness.
- Preserve unrelated user changes.
- Do not force push unless explicitly requested and verified.

## PR Body

Use:

- Summary
- Checks
- Risks
